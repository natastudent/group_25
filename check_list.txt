Чек-лист - список проверок.

структура чек-листа
1. Версия сборки (build), окружение, на котором проводилось тестирование)
2. Дата проведения теста 
3. Имя тестировщика
4. Тип тестов, которые были провдены
5. Название самих проверок
6. Результаты тестирования

названия столбцов для таблицы
-индентификатор (номер кейса)
-приоритет/вид теста, используемыйй в данной проверке
-ссылка на требования, с которыми связана проверка
-назваине модуля
-заглавие (отображает суть проверки)
-ожидаемый результат
-фактический результат
-номер дефекта для отслеживания в системе
